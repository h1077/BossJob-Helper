# AI 求职对话助手 — 设计文档

**日期**: 2026-05-31  
**状态**: 设计中  
**目标**: 在 Boss_helper.js 中新增 AI 与公司人事的智能沟通功能，从求职者角度出发，通过阶段式对话策略顺利拿到面试 offer。

---

## 1. 需求概述

在现有的 AI 自动回复基础上，构建完整的 AI 求职对话助手：
- 全流程覆盖（打招呼 → 首次回复 → 多轮对话 → 面试邀约）
- 从 DOM 实时提取对话上下文（轻量，不额外存储对话历史）
- 用户可选的三种对话策略（激进/平衡/保守）
- 5 阶段对话模型驱动 AI 回复

---

## 2. 核心架构

在 `Core` 对象中新增 `ConversationStrategy` 模块，包含三个子组件：

| 组件 | 职责 |
|------|------|
| `PhaseDetector` | 从 HR 最新消息中检测对话阶段 |
| `PromptBuilder` | 根据阶段 + 策略生成 prompt 模板 |
| `StrategyManager` | 管理策略配置和 UI 交互 |

### 数据流

```
HR 消息 → PhaseDetector(阶段检测) → PromptBuilder(阶段+策略→prompt) → requestAi() → 发送回复
```

### 集成点

- `Core.generatePersonalizedReply()` — 替换为新的阶段感知 prompt 系统
- `Core.aiReply()` — 直接使用新系统
- `HRInteractionManager._handleFirstInteraction()` — 打招呼阶段使用 Phase 1 策略
- 设置面板 — 新增策略选择器 UI

## 3. 5 阶段对话模型

### 3.1 阶段检测（PhaseDetector）

根据 HR 最新消息的关键词判断当前阶段：

| 阶段 | 名称 | HR 典型消息 | 触发关键词 |
|------|------|------------|-----------|
| Phase 1 | 打招呼破冰 | （刚投递，HR 尚无消息） | 默认阶段 |
| Phase 2 | 简历亮点展示 | "发个简历看看"、"介绍一下自己"、"你有什么经验" | 简历、介绍、经验、经历、背景、技能 |
| Phase 3 | 答疑解惑 | "为什么离职"、"住哪里"、"学历是"、"期望薪资多少" | 离职、为什么、住、学历、期望、加班、到岗 |
| Phase 4 | 薪资谈判 | "期望多少"、"薪资范围" | 薪资、工资、待遇、k、K |
| Phase 5 | 确认面试 | "方便面试吗"、"什么时候有空"、"来公司聊聊" | 面试、聊聊、有空、面谈、过来、时间 |

未匹配到任何阶段时，回退到通用对话模式。

### 3.2 每阶段 Prompt 设计

#### Phase 1 — 打招呼破冰
```
你是求职者，刚向HR投递了简历。请生成一条简洁的首次打招呼消息。
要求：30-50字，展示核心优势，表达对岗位的兴趣，语气友好不卑不亢。
{简历信息} {岗位名称} {策略指令}
```

#### Phase 2 — 简历亮点展示
```
你是求职者，HR要求你介绍自己/发简历。
请基于简历信息，用3-4句话展示：最匹配该岗位的技能、最亮眼的项目/工作经历、与岗位的契合点。
{简历信息} {岗位名称} {策略指令}
```

#### Phase 3 — 答疑解惑
```
你是求职者，HR在询问一些具体问题。请根据简历信息诚实回答。
如果是关于离职原因：正面表达职业规划。如果是住址/到岗时间：表示便利/尽快。
{简历信息} {HR消息} {策略指令}
```

#### Phase 4 — 薪资谈判
```
你是求职者，HR在谈论薪资。{策略指令}
{简历信息} {HR消息}
```

#### Phase 5 — 确认面试
```
你是求职者，HR发出了面试邀约。请积极确认，表示对面试机会的重视。
同时确认具体时间、形式（线上/线下）、需要准备什么材料。
{HR消息} {策略指令}
```

## 4. 策略系统

### 4.1 三种策略

| 策略 | 语气指令（注入到每阶段 prompt 中） | UI 说明文案 |
|------|--------------------------------|-----------|
| **激进** (aggressive) | 主动推进对话、适度包装优势、对不确定的问题给出自信回答、主动引导到面试环节、给出有竞争力的薪资期望 | 主动出击型：AI 会积极展示优势、适度包装经验、主动推进面试邀约，适合急需拿到 offer 的情况 |
| **平衡** (balanced) | 基于简历真实信息、积极但有分寸、不回避问题但也不过度承诺、适时推动面试、给出合理薪资范围并表示可协商 | 稳扎稳打型：AI 会如实展示能力、有技巧地回答问题、适时推动面试，适合大多数求职场景（默认） |
| **保守** (conservative) | 严格基于简历、对不确定的问题诚实回答、不主动推动但保持友好、尊重公司的薪资体系、以真诚换取信任 | 谨慎稳妥型：AI 会严格基于真实信息、不夸大不承诺、以真诚换取信任，适合对岗位匹配度有信心的场景 |

### 4.2 存储

- `localStorage` key: `conversationStrategy`
- 可选值: `"aggressive"` | `"balanced"` | `"conservative"`
- 默认值: `"balanced"`

## 5. UI 设计

在设置对话框的"聊天设置"标签页中，AI 角色设定下方新增策略选择器。

### 布局

使用三个 radio-style 卡片，每个包含：
- 单选按钮
- 策略名称
- 一行说明文字

默认选中"平衡"。

### 实现位置

在 `createSettingsDialog()` → `aiSettingsPanel` 中，紧跟 AI 角色设定 `roleSetting` 之后插入。

## 7. 不做什么

- 不存储完整对话历史到 localStorage（仅从 DOM 读取）
- 不对 AI 返回内容做 JSON 结构化解析（保持纯文本回复）
- 不新增外部依赖
- 不修改 `manifest.json` 中的权限配置
- 不改变现有打招呼语（greetingsList）的发送逻辑
